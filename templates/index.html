üìö How to Add MCP Servers

×

The MCP Client supports connecting to multiple MCP servers simultaneously, just like Claude Desktop!

üöÄ Quick Start

By default, clicking "Connect to Server" connects to the built-in example server with weather and math tools.

‚ûï Adding MCP Servers

You can add both local and remote MCP servers. Use the REST API endpoints:

üìÅ Local Servers (Running on Your Machine)

Example 1: Local Python Server

curl -X POST http://localhost:5001/servers/add \
  -H "Content-Type: application/json" \
  -d '{
    "name": "weather",
    "command": "python3",
    "args": ["/path/to/weather_server.py"]
  }'

Example 2: Local Node.js Server

curl -X POST http://localhost:5001/servers/add \
  -H "Content-Type: application/json" \
  -d '{
    "name": "nodejs-server",
    "command": "node",
    "args": ["/path/to/server.js"]
  }'

Example 3: Local npx Command

curl -X POST http://localhost:5001/servers/add \
  -H "Content-Type: application/json" \
  -d '{
    "name": "filesystem",
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/directory"]
  }'

üåê Remote Servers (Running on Another Machine)

Example 4: Remote Server via SSH

curl -X POST http://localhost:5001/servers/add \
  -H "Content-Type: application/json" \
  -d '{
    "name": "remote-db",
    "command": "ssh",
    "args": ["user@remote-server.com", "python3", "/remote/path/server.py"]
  }'

Example 5: Remote Server via HTTP/SSE

Note: For HTTP/SSE servers, use the SSE transport (requires additional configuration)

curl -X POST http://localhost:5001/servers/add \
  -H "Content-Type: application/json" \
  -d '{
    "name": "remote-api",
    "command": "python3",
    "args": ["-m", "mcp.client.sse", "https://api.example.com/mcp"]
  }'

Example 6: Docker Container Server

curl -X POST http://localhost:5001/servers/add \
  -H "Content-Type: application/json" \
  -d '{
    "name": "docker-server",
    "command": "docker",
    "args": ["run", "-i", "my-mcp-server-image"]
  }'

üîå Connecting to Servers

  • Connect to All: Click "Connect to Server" button
  • Connect to Specific: 
    curl -X POST http://localhost:5001/servers/connect \
  -H "Content-Type: application/json" \
  -d '{"name": "weather"}'

üìã Managing Servers

  • List Servers: curl http://localhost:5001/servers
  • Remove Server: 
    curl -X POST http://localhost:5001/servers/remove \
  -H "Content-Type: application/json" \
  -d '{"name": "weather"}'
  • Disconnect: 
    curl -X POST http://localhost:5001/servers/disconnect \
  -H "Content-Type: application/json" \
  -d '{"name": "weather"}'

üõ†Ô∏è How It Works

  1. Add multiple MCP servers with different tools
  2. Connect to all of them
  3. OpenAI sees all tools from all servers combined
  4. When OpenAI calls a tool, the client automatically routes it to the correct server
  5. View logs to see which server handles each request

üí° Example Use Case

You: "What's the weather in NYC and show me my database records"

What happens:

  • OpenAI calls get_weather ‚Üí routed to Weather Server
  • OpenAI calls query_database ‚Üí routed to Database Server
  • Both execute and return results in one conversation!
